# itb_stretch (髂胫束拉伸) 到 iOS 检测项映射规范

## 文档目的
- 基于训练产物 `config/action_configs/itb_stretch_static_config.json`，给出可直接执行的 iOS 迁移方案。
- 明确冠状面（正面）拉伸动作在“单侧/双侧”和“朝向抹平”逻辑上的特殊处理。
- 给出可用于后端人工配置的 JSON 示例。

## 动作业务描述与覆盖结论

**髂胫束拉伸 (itb_stretch)** 是一项**冠状面（正面）不对称发力的静态拉伸动作**。
它的标准姿态是：双腿交叉（一条腿在后并伸直，另一条腿在前并微曲），然后躯干向侧面大幅度弯曲，骨盆随之侧推。
虽然业务上这是一个单侧动作（有“拉伸左侧 ITB”和“拉伸右侧 ITB”的区别），但在 iOS 端，**我们依然无需拆分左腿和右腿两套配置**！通过对冠状面指标强制开启 `needs_absolute=1.0`（取绝对值），一套检测参数即可完美兼容向左弯曲和向右弯曲的所有情况。

## 关键边界与协议说明

### 关于 5 槽位协议的兼容性
我们在前几个单腿动作中引入了 `[min, max, needs_absolute, debounce, side_selection_mode]` 的 5 槽位协议。
对于髂胫束拉伸：
- 它的两个核心指标 `trunk_lateral_flexion` 和 `pelvic_obliquity` 都是**全局中心点指标**（计算左右肩/胯的中点与地面的夹角），**根本不涉及左右侧选腿**。
- 它的 `knee_flexion` 虽然是双侧指标，但由于交叉腿的特殊性，两条腿中必然有一条较直（后侧拉伸腿），一条较弯（前侧支撑腿）。因此，我们直接复用找大角度腿（更直的腿）的逻辑即可，即 **`side_selection_mode = 2.0`**。

### iOS 复用判定与分析（100% 复用！）

- `trunk_lateral_flexion` (躯干侧倾角) -> **复用 `itemID=44` (Trunk Lateral Flexion Static Hold)**。
  - 在“单脚站立”中我们已规划该 ID。当时要求躯干不侧倾（接近 0°），这里要求大幅度侧倾（> 3°）。通过传入不同的阈值直接复用，第 5 槽位填 `0.0`。
- `pelvic_obliquity` (骨盆侧倾/顶胯角) -> **复用 `itemID=43` (Pelvic Obliquity Static Hold)**。
  - 在“单脚站立”中已规划该 ID。直接复用，第 5 槽位填 `0.0`。
- `knee_flexion` (膝关节屈曲) -> **复用 `itemID=46` (Knee Flexion Static Hold)**。
  - 我们需要找到被拉伸的那条腿（在后面伸直的腿），由于它比交叉在前面的腿更直，因此第 5 槽位下发 **`2.0`**（寻找角度最大的一侧）。

## 逐字段 -> 逐索引映射

### itemID=44（Trunk Lateral Flexion，躯干侧倾角）- 复用
- 来源字段：
  - `normal_range` = `[3.15, 50.43]` (必须侧倾，上限放宽以容纳柔韧性极好的人)
  - `needs_absolute` = `1.0`

| iOS item44 索引 | 来源字段 | 说明 |
|---|---|---|
| `[0]` | `3.15` | normal_range 下界 (必须有至少 3.15 度的明显侧倾) |
| `[1]` | `50.43` | normal_range 上界 |
| `[2]` | `1.0` | **开启绝对值处理 (抹平向左弯和向右弯的差异)** |
| `[3]` | `15.0` | 0.5秒防抖 |
| `[4]` | `0.0` | 全局中心指标，不涉及选腿 |

**建议参数行：**
- `[3.15, 50.43, 1.0, 15.0, 0.0]`

---

### itemID=43（Pelvic Obliquity，骨盆侧倾角）- 复用
- 来源字段：
  - `normal_range` = `[-14.75, 19.14]` (转换为绝对值后为 `[0.0, 19.5]`)
  - `needs_absolute` = `1.0`

| iOS item43 索引 | 来源字段 | 说明 |
|---|---|---|
| `[0]` | `0.0` | normal_range 下界 |
| `[1]` | `19.5` | normal_range 上界 |
| `[2]` | `1.0` | **开启绝对值处理** |
| `[3]` | `15.0` | 0.5秒防抖 |
| `[4]` | `0.0` | 全局中心指标，不涉及选腿 |

**建议参数行：**
- `[0.0, 19.5, 1.0, 15.0, 0.0]`

---

### itemID=46（Knee Flexion，膝关节屈曲）- 复用
- 来源字段：
  - `normal_range` = `[100.34, 158.35]` (上限放宽至 185.0，允许后腿完全伸直或微曲；下限 100° 是为了容忍交叉腿的姿态干扰)
  - `needs_absolute` = `0.0`

| iOS item46 索引 | 来源字段 | 说明 |
|---|---|---|
| `[0]` | `100.34` | normal_range 下界 |
| `[1]` | `185.0` | normal_range 上界 (允许后腿完全伸直) |
| `[2]` | `0.0` | 关闭绝对值处理 |
| `[3]` | `15.0` | 0.5秒防抖 |
| `[4]` | `2.0` | **指示策略强制寻找两腿中更直的那一条腿作为主导腿** |

**建议参数行：**
- `[100.34, 185.0, 0.0, 15.0, 2.0]`

## 外部配置示例（可人工配置）

### 完整覆盖（含躯干侧弯、骨盆顶出、后腿不严重弯曲）
```json
{
  "detectItemIDs": ["44", "43", "46"],
  "detectItemParameters": [
    [3.15, 50.43, 1.0, 15.0, 0.0],
    [0.0, 19.5, 1.0, 15.0, 0.0],
    [100.34, 185.0, 0.0, 15.0, 2.0]
  ]
}
```

## 已可直接上线项 vs 需新增开发项
- **已可直接上线项**：无。
- **需新增开发项**：
  - 前置依赖：必须完成在“股四头肌拉伸”和“腘绳肌拉伸”中规划的 `side_selection_mode` 5 槽位寻边状态机。一旦该底层设施完工，本动作的三个 ID 均可零代码直接接入！

## 验收点与防作弊/熔断逻辑
1. **站直不弯腰熔断 (Trunk Lateral Flexion)**：用户如果仅仅双腿交叉站立，但是身体不向侧面弯曲去感受拉伸（躯干侧倾角小于 3.15 度），`itemID=44` 会触发拦截，提示“上身向侧面弯曲”。
2. **极度弯膝熔断 (Knee Flexion)**：虽然允许拉伸腿因为柔韧性不足而微曲，但如果用户直接处于蹲姿或两腿都严重弯曲（小于 100度），完全失去了拉伸意义，此时 `itemID=46` 会触发拦截。
3. **左右无感兼容验收**：测试人员分别进行左侧 ITB 拉伸和右侧 ITB 拉伸。由于对冠状面指标全部开启了 `needs_absolute=1.0`，系统必须能够同时兼容这两种方向的拉伸，而无需切换检测项。
