# side_stretch (侧腰拉伸) 到 iOS 检测项映射规范

## 文档目的
- 基于训练产物 `config/action_configs/side_stretch_static_config.json`，给出可直接执行的 iOS 迁移方案。
- 延续冠状面动作的 `needs_absolute=1.0` 抹平策略。
- 给出可用于后端人工配置的 JSON 示例。

## 动作业务描述与覆盖结论

**侧腰拉伸 (side_stretch)** 是一项**冠状面（正面）对称动作（向两侧交替进行）**。
标准的侧腰拉伸动作主要在冠状面进行，要求双脚站立或分开，躯干向一侧大幅度弯曲，同时骨盆向对侧顶出，以拉伸对侧的侧腹部和背阔肌。
虽然业务上用户需要分别向左侧和右侧拉伸，但在 iOS 端，**我们无需拆分左弯和右弯两套配置**。通过对冠状面的核心指标强制开启 `needs_absolute=1.0`（取绝对值），一套检测参数即可完美兼容向左弯曲和向右弯曲的所有情况。

## 关键边界与协议说明

### iOS 复用判定与分析（100% 复用！）

我们在上一个“髂胫束拉伸（ITB Stretch）”动作中，已经为冠状面的两个核心指标规划了对应的静态 ID。对于侧腰拉伸，我们直接复用它们，只需替换为刚刚提取的、更符合侧腰拉伸紧绷度的严格参数即可。

- `trunk_lateral_flexion` (躯干侧倾角) -> **复用 `itemID=44` (Trunk Lateral Flexion Static Hold)**。
  - 核心指标。评估上身向侧面弯曲的幅度。全局中心点指标，不涉及选腿，第 5 槽位填 `0.0`。
- `pelvic_obliquity` (骨盆倾斜角) -> **复用 `itemID=43` (Pelvic Obliquity Static Hold)**。
  - 辅助指标。评估由于上身侧倾带来的骨盆代偿性倾斜。全局中心点指标，第 5 槽位填 `0.0`。

*(注：在原始配置中，曾有 `trunk_lean` 指标。但因 2D 姿态估计中前倾和侧倾在纯正面朝向时产生数学坍缩，该冗余指标已被移除，仅依靠侧倾即可胜任判定。)*

## 逐字段 -> 逐索引映射

### itemID=44（Trunk Lateral Flexion，躯干侧倾角）- 复用
- 来源字段：
  - `normal_range` = `[12.32, 25.06]` (侧腰拉伸幅度有限，上限被卡在 25° 左右，高于此值可能是在做下腰动作)
  - `needs_absolute` = `1.0`

| iOS item44 索引 | 来源字段 | 说明 |
|---|---|---|
| `[0]` | `12.32` | normal_range 下界 (必须有至少 12.32 度的明显侧倾) |
| `[1]` | `30.0` | normal_range 上界 (适当放宽至 30度，容纳柔韧性好的人) |
| `[2]` | `1.0` | **开启绝对值处理 (抹平向左弯和向右弯的差异)** |
| `[3]` | `15.0` | 0.5秒防抖 |
| `[4]` | `0.0` | 全局中心指标，不涉及选腿 |

**建议参数行：**
- `[12.32, 30.0, 1.0, 15.0, 0.0]`

---

### itemID=43（Pelvic Obliquity，骨盆侧倾角）- 复用
- 来源字段：
  - `normal_range` = `[11.51, 31.5]` (骨盆必须有明显的倾斜和顶出)
  - `needs_absolute` = `1.0`

| iOS item43 索引 | 来源字段 | 说明 |
|---|---|---|
| `[0]` | `11.51` | normal_range 下界 (骨盆必须倾斜至少 11.51 度) |
| `[1]` | `35.0` | normal_range 上界 (适当放宽至 35.0) |
| `[2]` | `1.0` | **开启绝对值处理** |
| `[3]` | `15.0` | 0.5秒防抖 |
| `[4]` | `0.0` | 全局中心指标，不涉及选腿 |

**建议参数行：**
- `[11.51, 35.0, 1.0, 15.0, 0.0]`

## 外部配置示例（可人工配置）

### 完整覆盖（含躯干侧弯、骨盆顶出）
```json
{
  "detectItemIDs": ["44", "43"],
  "detectItemParameters": [
    [12.32, 30.0, 1.0, 15.0, 0.0],
    [11.51, 35.0, 1.0, 15.0, 0.0]
  ]
}
```

## 已可直接上线项 vs 需新增开发项
- **已可直接上线项**：由于这两个 ID 在“单脚站立”和“髂胫束拉伸”中已经规划，且不涉及复杂的 5 槽位寻腿状态机（全部下发 0.0），本动作完全可以直接复用上线。
- **需新增开发项**：无。

## 验收点与防作弊/熔断逻辑
1. **不侧倾熔断 (Trunk Lateral Flexion)**：用户如果只是站着，或者只弯了一点点（侧倾角小于 12.32 度），拉伸幅度不够，`itemID=44` 会触发拦截，提示“上身向侧面弯曲”。
2. **骨盆不动熔断 (Pelvic Obliquity)**：如果用户只是头和肩膀侧弯，但胯部死死锁住不动（倾斜小于 11.51 度），说明拉伸没有延展到侧腰，`itemID=43` 会触发拦截，提示“跨部向侧面顶出”。
3. **左右无感兼容验收**：测试人员分别进行向左侧腰拉伸和向右侧腰拉伸。由于对冠状面指标全部开启了 `needs_absolute=1.0`，系统必须能够同时兼容这两种方向的拉伸，而无需切换检测项。
